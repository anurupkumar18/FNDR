3D Neural Network Design Overview

Hybrid orbital-community layout with local force simulation

Architecture Diagram
LEVEL 1: Active Query/Context
Central sphere representing current query, project, or memory focus
LEVEL 2: Semantic Communities (Orbital Rings)
• Work/Code, Research, Design, Meetings
• Errors/Debugging, People, Files, Decisions
• Todos, Concepts, Past Searches, Agent Context
→ Stable 3D positions in spherical arrangement
LEVEL 3: Memory Cards & Entities (Local Clusters)
Lightweight force simulation inside each community
→ Nodes repel within their region, attract to region anchor
LEVEL 4: Evidence/Provenance (On Hover/Click)
Source memories, confidence scores, relationships
→ Progressive disclosure, not cluttering the base graph
LEVEL 5: Agent Actions (Side Panel)
Send to agent, refine, filter, export
→ Separate from graph visualization
Z-Depth Semantics
Z forward (toward camera):
• High query relevance
• High importance (frequently used)
• Recent memories

Z backward (away from camera):
• Low query relevance
• Low importance (rarely used)
• Old memories (drift back unless reused)

Smooth transitions as relevance and time change
Visual Behaviors

On Search/Query: Relevant clusters move forward and brighten. Unrelated fade back but stay visible.

Node Sizing: Proportional to importance, frequency of access, or connection count.

Node Color: Community-based (semantic region color), with brightness by importance.

Edges: Sparse by default (only strongest connections). Progressive reveal on hover/filter.

Interaction: Orbit/pan/zoom feels like moving through a brain map. Dragging is smooth.
